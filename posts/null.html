Hexo+GitHub搭建个人博客

博客 Hexo Github
博客篇
发布日期:   2021-12-08
更新日期:   2022-03-04
文章字数:   1.2k
阅读时长:   4 分

1. Git和Github

1.1 Github账户创建和仓库创建

GitHub官网：https://github.com/

在github网页上创建一个以http://username.github.io命名的repositories,此时username为自己github的账号名称。

新建一个名为用户名.github.io的仓库，例如，如果你的github用户名是VictorWayne，那么你就新建VictorWayne.github.io的仓库（必须是你的用户名，其它名称无效），将来你的网站访问地址就是VictorWayne.github.io，弊端就是，我们只能创建一个这样的仓库。

1.2 Git的安装的配置

Git和Github的使用：https://blog.csdn.net/u013162035/article/details/78476880

Git下载：https://git-scm.com/downloads

Git安装：https://blog.csdn.net/u013162035/article/details/78464161

安装完成后在命令提示符中输入git –version验证是否安装成功。

首先右键打开git bash，然后输入下面命令：

git config --global user.name "VictorWayne"
git config --global user.email "yunlong_wei@outlook.com"

用户名和邮箱根据你注册github的信息自行修改。

然后生成密钥SSH key：

ssh-keygen -t rsa -C "yunlong_wei@outlook.com"

然后一直回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容。
或者在git bash中输入：

cat ~/.ssh/id_rsa.pub

复制输出的内容。

打开你的github主页，在头像下面点击settings，进入个人设置，再点击SSH and GPG keys，再点击New SSH key，新建一个SSH，将刚复制的内容粘贴到key那里，title随便填，保存。

输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了。

2. 环境的搭建

2.1 安装Node.js

Node.js官方网站：https://nodejs.org/en/

Node.js官方文档：https://nodejs.org/en/docs/

Node.js安装文档：https://nodejs.org/en/download/
Hexo是基于node.js的，所以我们在安装它之前需要用到npm安装工具，这个工具是 node.js 安装包的工具，所以，我们先要安装 node.js。

笔者是用的Windows 64位的，到node.js下载安装包。
安装选项全部默认，一路点击Next。

最后安装好之后，按Win+R打开命令提示符，输入node -v和npm -v，如果出现版本号，那么就安装成功了。

2.2 安装Hexo

Hexo官方：https://hexo.io/

Hexo官方(中文)：https://hexo.io/zh-cn/
官方手册：https://hexo.io/zh-cn/docs/inde
在合适的地方新建一个文件夹，用来存放自己的博客文件，比如我的博客文件都存放在D:\study\program\blog目录下。

在该目录下右键点击Git Bash Here，打开git的控制台窗口，以后我们所有的操作都在git控制台进行，就不要用Windows自带的控制台了。
定位到该目录下，输入以下命令安装Hexo。

npm i hexo-cli -g

# npm install -g hexo-cli
# npm install hexo-cli -g
# npm install -g hexo

会有几个报错，无视它就行。

安装完后输入hexo -v验证是否安装成功。

2.3 使用Hexo进行本地建站

输入

hexo init

# hexo init blog

初始化文件夹
接着输入

npm install

安装必备的组件。
这样本地的网站配置也弄好啦，输入

hexo g

# hexo generate

生成静态网页

为了测试本地建站是否成功，输入

hexo s

# hexo server

打开本地服务器，然后浏览器打开http://localhost:4000/，就可以看到我们的博客了

2.4 部署到Github上

在这里我们使用的是git源码管理工具，所以，我需要安装git包进行部署，安装这个插件才能使用git进行自动部署。

npm install hexo-deployer-git -save

在接下来，我们将要生成网站了，首先清理一下缓存。

hexo clean

当我们部署网站前，需要先生成静态网站。它会自动在目录下创建public的目录, 并将新生成的网页存放在这个目录里。

hexo g
# hexo generate

注意部署前需要重新生成网站, 每一次修改后都需要重新生成网站并进行部署，生成网站前先生成静态网站。
输入

hexo d

# hexo deploy

如果在部署出现错误信息如下，则需要安装git插件。

npm install hexo-deployer-git -save

以上也可以使用一步就搞定：

hexo g -d

上述的部署指令中hexo deploy可以换成hexo server，两者的区别在于，前者是将博客部署到远程的Github上，而后者是运行在本地，通过http://localhost:4000在浏览器中访问。后者是为了调试配置方便而使用，但是最终本地博客还是需要`hexo deploy`指令将其部署至Github上。

deploy:
  type: git
  repository: git@github.com:liuhuanhuan963019/liuhuanhuan963019.github.io.git
  branch: master
